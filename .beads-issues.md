**Status:** Open

```python
inv = Inventory()
inv.add_item("Sword", "A blade", quantity=1, weight=5)
items = inv.list_items()

```

**Expected Behavior:** Return dict[str, Item] mapping names to items

# BUG-009: WebUI Inventory Sidebar Shows (empty) Despite Item Pickup

**Severity:** P3
**Status:** Closed (fixed in v0.8.1)

After the narrator describes the player picking up an item, the sidebar inventory still reads
`(empty)`. The `renderState()` call expects `inventory.items` to be a list, but `Inventory.to_dict()`
serializes items as a dict keyed by name. The JS check `Array.isArray(inv.items)` fails on
the dict, so it falls through to the empty display.

**Location:** rag_quest/web/static/index.html — `renderState()` inventory section (~line 366)

**Expected Behavior:** Inventory sidebar should display any items present in the serialized state.

# FEATURE-001: WebUI Game Onboarding Flow

**Severity:** P2
**Status:** Closed (implemented in v0.8.2)

The WebUI currently requires a pre-existing save file to start playing. New players who visit
the web client see a save selector with no entries and have no way to create a character or
world. The interactive TUI onboarding (setup wizard with Fresh Adventure / Quick Start / Upload
Lore modes, character creation, and Ollama auto-detection) is only available in the CLI via
`rag_quest/startup.py`, `config.py::setup_first_run()`, and `engine/game.py::run_game()`.

The web client needs an equivalent onboarding flow so a player can go from zero to adventuring
without ever touching the terminal.

**Scope:**
- New `/session/new` API endpoint(s) that drive character creation + world setup
- WebUI screens: provider selection (auto-detect Ollama), start mode (Fresh / Quick Start /
  Upload Lore), character creation (name, race, class with stat previews), confirmation
- Reuse existing `setup_first_run()` logic on the backend where possible; factor out the
  interactive prompts so the same validation works for both TUI and web
- Template selection should surface the 4 built-in templates from `worlds/templates.py`
- Lore upload via file input (PDF/txt/md) through a new upload endpoint

**Location:** `rag_quest/web/app.py` (endpoints), `rag_quest/web/static/index.html` (UI)

**Dependencies:** None — additive feature, no existing code changes required.

**References:** `rag_quest/startup.py`, `rag_quest/config.py::setup_first_run()`,
`rag_quest/engine/character.py` (races, classes, stat bonuses), `rag_quest/worlds/templates.py`
